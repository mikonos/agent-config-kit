## 0) 关键结论（先记住这 3 条）
- **比例**：图文轮播优先 **3:4**（常用 **1080×1440** / 1242×1656）
- **结构**：每页「**1 个结论 + 3–5 条要点**」，要点短句化（收藏友好）
- **出图清晰度**：用 **2x 渲染**（文字更"利"）

## 1) 轮播内容结构（默认 8 页）
- **P1 封面**：数字化承诺 + 关键词（"3 秒说明白"）
  - 例：`从提示词矩阵到 Skills：稳定性提升的 3 个关键`
- **P2 痛点**：你是不是也这样？（共鸣）
- **P3 结论**：一句话讲清"为什么"
- **P4 方法**：框架/系统（给读者一个"抓手"）
- **P5 步骤**：1-2-3（可照做）
- **P6 示例**：一个具体例子（减少抽象）
- **P7 清单**：Checklist（收藏页）
- **P8 CTA**：收藏 + 评论（收集场景）

## 2) 视觉排版准则（小红书更"像"的那套）

### 2.1 色彩与风格（默认推荐）
- **白底/浅底** + **1 主色** + **1 辅色**（总色不超过 3）
- 少用大面积深色背景（更像"开发者文档/Keynote"，不利于"红薯感"）

### 2.2 信息密度
- 目标：留白 **30%–40%**（不要出现"下半屏大片空白"）
- 每页只讲一件事：标题 ≠ 段落；段落拆成要点

### 2.3 字号/行距（经验值，可调）
- 标题：64–80（封面可以更大）
- 正文：30–34
- 注释/页脚：22–24
- 行距：1.25–1.4

### 2.4 强调方式（比"加粗"更有效）
- **色块高亮关键词**（mark）
- **贴纸/角标**：可收藏、页数、角色标签（痛点/步骤/清单）
- **卡片化**：列表/对照表/步骤卡片比长段更像小红书

## 3) 工程化出图

### 3.1 前提约定（HTML）
- 每页一个 `.page` 固定尺寸 1080×1440
- **所有 `.page` 都保持 `display: flex`**（不要用 `display: none` + `:target` 切换）
- 内容垂直居中：`justify-content: center`

### 3.2 推荐方案：Playwright 截图（稳定可靠）

```bash
# 1. 在 HTML 所在目录安装 Playwright（首次）
npm init -y && npm install playwright

# 2. 运行出图脚本
node "<installed-skill-dir>/scripts/render_deck_playwright.js" \
  "20260128_小红书轮播_xxx.html" \
  --output-dir "20260128_xxx_png" \
  --prefix "20260128_xxx"
```

**为什么推荐 Playwright**：
- 直接截取每个 `.page` 元素，不依赖 CSS hash / `:target`
- 默认 2x 渲染（`deviceScaleFactor: 2`），文字清晰锐利
- 自动等待 PlantUML 等外部图片加载完成
- 稳定性高，不会出现"P2–P8 空白/黑屏"问题

### 3.3 备选方案：Chrome headless（不推荐）

> ⚠️ 此方案依赖 CSS `:target` 选择器，在 headless 模式下行为不稳定，可能导致 P2–P8 截图为空白。

如果仍要使用，需要在 HTML 中添加：
```css
.page { display: none; }
.page:first-of-type { display: flex; }
.page:target { display: flex; }
```

示例（macOS）：
- 渲染 2x：`--force-device-scale-factor=2`（得到 2160×2880）
- 再缩回：`sips -Z 1440` → 1080×1440（保持 3:4）

### 3.4 关键 CSS 模板（Playwright 友好）

```css
body {
  margin: 0;
  padding: 0;
  background: #ffffff;
}
.page {
  width: 1080px;
  height: 1440px;
  background: #ffffff;
  display: flex;
  flex-direction: column;
  justify-content: center;  /* 内容垂直居中，避免头重脚轻 */
  padding: 72px 96px 80px;
  overflow: hidden;
}
.content {
  /* ⚠️ 关键：不要加 flex: 1，否则会被拉伸占满空间 */
  display: flex;
  flex-direction: column;
  gap: 24px;
}
```

**居中关键点**：
- `.page` 用 `justify-content: center` 实现垂直居中
- `.content` **不要设置 `flex: 1`**（否则会占满剩余空间，导致内容被拉伸而不是居中）
- 让 header / content / footer 自然堆叠，整体在 `.page` 中居中

## 4) 质量检查（发布前 30 秒过一遍）
- [ ] **封面 3 秒说明白**：主题 + 结果承诺 + 关键词（最好带数字）
- [ ] **单页不超载**：要点 ≤ 5；标题 ≤ 2 行
- [ ] **术语密度可读**：不懂的人也能看懂"结论"
- [ ] **主 CTA 单一**：默认"收藏"（评论用于收集场景）
- [ ] **截图尺寸一致**：1080×1440，页码顺序不乱
- [ ] **P2–P8 不是空白**：如果空白，改用 Playwright 方案

---

## 5) 常见失败模式（直接避坑）
- **像 PPT**：颜色太暗/留白太大/强调方式只有加粗 → 改成白底卡片 + 贴纸 + 色块重点
- **像长文**：一页一段话 → 拆成「结论 + 要点」
- **不够利**：截图发灰/糊 → 用 2x 渲染
- **P2–P8 空白/黑屏**：不要用 CSS `:target` 控制显示 → 改用 Playwright 直接截取 `.page` 元素
- **内容不居中**：`.page` 加 `justify-content: center`
