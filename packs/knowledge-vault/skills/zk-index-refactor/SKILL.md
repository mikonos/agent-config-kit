# ZK Index Refactor

把 `03_索引/` 从“平铺索引堆”收口成“关键词总表 + 少量树干 Hub + 项目子层”。

先确认目标 Vault 的索引根目录；`03_索引/` 只是默认示例。默认只输出去留表和改链预览。移动、批量改链或删除文件前，必须展示精确文件清单并获得用户确认；删除任何文件都要单独确认一次。

## 不适用场景

以下情况不要优先用这个 skill：

- 只是给单篇笔记入网、补双链、挂一个索引入口
- 只是改某篇索引的局部内容，没有根目录层级问题
- 只是做项目资料搬运，不涉及 `03_索引/` 的结构治理

## 先读什么

在这个 vault 里，先读：

- `03_索引/README_索引整理与卢曼视角.md`
- `03_索引/00_索引_关键词总表.md`
- `references/checklist.md`

如果是承接一轮未完成的索引治理，再读当天执行计划或去留表。

如果需要直接落盘，读：

- `references/output-templates.md`

## 什么时候用

- 根目录索引文件太多，已经分不清树干和枝叶
- 某批子专题明显应该并回父 Hub
- 项目型索引、操作指南、需求台账混进了 `03_索引/` 根目录
- 需要把旧 wikilink 改到父 Hub 标题锚点，避免删文件后断链
- 需要把这轮动作沉淀成“可继续接手”的中控台

## 核心判断

不要先问“它放哪个分类”，先问：

1. 它是否回答一个独立且稳定的查找问题？
2. 它是否覆盖多个原子笔记或多个子主题？
3. 删掉它会失去一个一级入口，还是只失去一个别名？
4. 它更自然地是根目录树干，还是某个父 Hub 的一个标题锚点？
5. 它本质上是不是项目/运营/执行文档，应该下沉到 `03_索引/<主题或项目>/`？

结论只允许四种：

- 保留为顶级 Hub
- 并入现有 Hub
- 移入项目目录
- 降级为过渡页

可以用这条速判：

- 能独立回答一个长期问题，保留
- 只是父 Hub 的一节，归并
- 本质是项目运营面板，下沉
- 还有历史检索价值但不再主导航，过渡页

## 根目录准入规则

根目录只留：

- `00_索引_关键词总表`
- `README_索引整理与卢曼视角`
- 真正的顶级 Hub

默认不留根目录的东西：

- 操作指南、执行计划、会议纪要、需求/验收台账、Demo 叙事、发布看板
- 过渡页、历史入口、重定向页
- 只是父 Hub 一节内容的子专题
- 单篇深度阅读衍生的窄索引

## 标准流程

### 1. 先建中控台

至少落两张：

- 一张“执行计划”
- 一张“根目录去留表”

去留表每个文件都要有明确去向，不允许“先删再想”。

模板见 `references/output-templates.md`。

### 2. 先收硬边界，再收语义边界

优先处理：

- 明显的重复路径
- 目的不明的目录
- 项目型文件误入根目录

然后再做真正的语义归并。

### 3. 按簇处理，不要见一个并一个

每次只打一簇，例如：

- Agent / Harness / 记忆
- 产品方法论
- AI时代观察
- 商业模式 / 融资 / 创始人
- OpenClaw / 项目运营

做完一簇再验收，不跨簇乱并。

### 4. 先补父 Hub 锚点，再改全库旧链

删除子索引前，必须先在父 Hub 中补稳定标题，例如：

- `## 第二轮归并（YYYY-MM-DD）`
- `### 子专题：<主题名>`

旧链接统一改写到父 Hub 标题锚点，例如：

- `[[父Hub#子专题：xxx|旧索引名]]`

先改链，后删文件。

如果旧索引只是换了目录但文件名不变，通常不需要全库改链，但要补新的项目总入口。

### 5. 物理收口只做两种

- 真并回：父 Hub 已承接、全库旧链已改完、再删子索引
- 真下沉：项目/运营型文件移入 `03_索引/<项目>/`

如果只是历史入口还有检索价值，就降级成过渡页，不继续占根目录。

### 6. 每轮都回写数字

至少回写：

- 根目录 `.md` 数量
- 空文件数量
- 本轮并回删除了多少子索引
- 本轮移走多少项目型文件
- 剩余待判断项还有哪些

## 操作纪律

- 复杂多步且可能不可逆时，先给出计划，再执行
- 只在“父 Hub 已承接 + 旧链已改写”且用户确认精确删除清单后删除索引文件
- 模糊项不要硬并，先记入待判断
- 不要把项目索引强塞进知识树干
- 不要制造空 `.md` 作为占位
- 根目录减少不是目标本身，权威入口唯一才是目标

## 推荐命令

优先用快速、可复查的方式：

- 列根目录：`rg --files 03_索引 | rg '^03_索引/[^/]+\\.md$'`
- 查空文件：`find 03_索引 -name '*.md' -size 0`
- 查旧索引残留：`rg -n '\[\[(旧索引名)(\||\]\])'`

如果需要批量改链，先做精确匹配，再删文件。

## 验收

至少做四项：

1. `03_索引` 根目录 `.md` 数量是否下降
2. 根目录空文件是否为 `0`
3. 本轮删掉的旧索引名，精确 wikilink 残留是否为 `0`
4. `README` 与 `00_索引_关键词总表` 是否补了新秩序规则
