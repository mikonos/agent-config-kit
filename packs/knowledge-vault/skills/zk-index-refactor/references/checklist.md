# ZK Index Refactor Checklist

## 开始前

- 读 `03_索引/README_索引整理与卢曼视角.md`
- 读 `03_索引/00_索引_关键词总表.md`
- 确认是否已有执行计划 / 去留表

## 分类

- 保留为顶级 Hub
- 并入现有 Hub
- 移入项目目录
- 降级为过渡页

## 先做

- 统计根目录 `.md` 数量
- 统计空 `.md` 数量
- 列出根目录所有 `.md`
- 标出明显项目型文件

## 并回父 Hub 的步骤

1. 给父 Hub 补“第二轮归并”段
2. 给每个子专题补稳定标题锚点
3. 全库改写旧 wikilink 到父 Hub 锚点
4. 精确检查旧索引名残留
5. 展示精确删除清单并获得用户确认后，删除子索引文件

## 下沉项目层的步骤

1. 判断它是否是项目/执行/运营文档
2. 移入 `03_索引/<主题或项目>/`
3. 如有必要，在父 Hub 或项目总 Hub 里补入口

## 收官验收

- 根目录 `.md` 数量是否下降
- 空文件是否为 `0`
- 已删除旧索引名残留是否为 `0`
- `README` 是否写明现行规则
- `关键词总表` 是否优先指向规范入口
