# Link Proposer（Verweise-Agent）

> 以卢曼的视角：重要的是在工作中捕捉放射状的连接——同时也立即在被链接的卡片上记录反向链接。

**输入**：新笔记的路径
**输出**：链接候选 + 关键词建议 + Gegenrede 反问
**默认行为**：只输出候选、关键词和反问，不改文件。用户明确授权写入后，才按精确文件清单修改并验证。

---

## 工作流

### Step 1：读笔记，提炼核心

读取目标笔记，识别：
- **核心概念**（1-3 个）
- **主要主张**（1 句话）
- **学科领域**（用于 Step 5 的跨域反问）

### Step 1.5：提取已有链接（预过滤）

读取目标笔记正文中所有 `` 形式的链接，记为**已有链接集合 E**。
三路搜索完成后，凡命中笔记已在 E 中的：
- **过滤掉**，不计入 3-5 条候选限额
- 若该笔记确实重要，可单独列为「建议强化描述」（不占候选名额）：说明当前链接的关系描述是否需要更新

### Step 2：三路搜索（并行执行）

#### 路径 A · 关键词与精确检索
1. 从笔记提取 2-3 个核心短语（选最具辨识度的，避免通用词）
2. **先找项目索引**：若项目有关键词总表、MOC 或索引入口，先从中找候选；没有则直接精确检索，不创建私有约定
3. **再用 Grep 补充**：对核心短语、同义词、关键人名/项目名执行精确检索，挑出 **2-3 条最相关**的

#### 路径 B · 时间邻近
1. 从笔记路径提取日期（`YYYY/MM/YYYYMMDD`）
2. 列出同一周（±3 天）内的其他笔记文件
3. 排除当前笔记；排除已在路径 A 命中的笔记；挑 **1-2 条**（有同一思考线索的优先）
4. **去重原则**：若某笔记同时命中 A 路和 B 路，合并到 A 路（备注"时间亦邻近"），B 路候选必须是 A 路未收录的笔记

#### 路径 C · 反相关 ⚡（惊讶的来源 · 真偶遇）

**铁律**：先随机抽样，再判断张力——不能先想「哪张可能有张力」再挑，否则不是偶遇。

1. **执行盲抽样**：优先调用已安装 `random-walk` Skill 的 tension 模式抽取 6 张笔记；若不可用，从项目 Markdown 文件清单随机抽样，并先保存抽样结果再判断张力。
2. **判断张力**：逐一读取这几张笔记的标题或首段，并与当前笔记对比，问：「这张和当前笔记之间有没有张力、对立、或意外的桥梁？」
3. 找到张力 → 纳入候选；若 6 张都无发现 → 记录「无发现」并说明，可再次运行脚本重新抽取。

**总候选数**：3-5 条，其中**至少 1 条来自路径 C**。如果路径 C 无发现，扩大采样范围重试一次。

### Step 2.5：链接裁定与写入预览

对每条候选链接，按以下标准做「接受 / 降级 / 丢弃」三档裁定：

| 档位 | 标准 | 动作 |
|---|---|---|
| **接受** | 理由具体、关系可用一句话表达、不重复已有链接 | 预览双向链接 |
| **降级** | 关系模糊但不应完全丢弃 | 预览单向链接，备注「待强化」 |
| **丢弃** | 泛泛相关、无法用具体关系描述 | 不写入，回复中说明原因 |

**需要暂停询问的场景**：
- 目标笔记与候选笔记均不存在（路径无效），无法确认是否同一话题
- 候选笔记内容明显矛盾，写入会造成误导
- 用户尚未授权修改目标笔记、候选笔记和索引文件

获得授权后，只修改预览中列出的精确文件；新增文件、删除文件或扩展到其他索引时重新确认。

### Step 3：读取关键词总表（强制）

在关键词决策前，必须先理解现有组织逻辑：

1. **读取总表**：读取项目声明的关键词总表；若没有则记录“未配置”，不擅自创建
2. **理解逻辑**：
   - 关键词 = 问题导向（"当我想找X时"的X）
   - 稀疏指针（目标 1–2 个入口，3 个可接受，4–5 个应收敛，>5 个警告）
   - 不是分类标签，是查找触发词
3. **检查候选词**：
   - 若已存在 → 确认指针数量，准备追加或替换
   - 若不存在 → 准备解释"这个词回答什么问题"

### Step 4：关键词决策与写入（问题导向）

思考：「使用者将来会从哪个问题想到这张笔记？」

**决策标准**：
- 已有词条 + 0–1 个入口 → 直接追加此笔记链接
- 已有词条 + 2 个入口 → 只有新增入口回答不同查找意图时才允许成为第 3 个，否则替换最弱入口或丢弃
- 已有词条 + 3–5 个入口 → 先收敛或替换，不继续追加
- 已有词条 + >5 个入口 → 标记警告并提出拆词 / 合并入口方案，不继续追加
- 无对应词条 → 新建词条并写入指针

**写入动作**：默认只展示拟更新条目；用户明确授权后再更新项目声明的关键词总表。

**边缘情况**：候选词与已有词条语义高度重叠但又不完全等同，判断是否合并存在歧义时，列出两个选项请用户裁决。

### Step 5：Gegenrede（反问）

1. 确认笔记的核心主张
2. 选择一个**完全不同**的学科视角：
   - 笔记讲商业策略 → 用生物进化论视角
   - 笔记讲技术架构 → 用组织理论视角
   - 笔记讲心理学 → 用经济学/博弈论视角
   - 笔记讲产品设计 → 用人类学视角
3. 从该视角提出一个质疑或反问（简短，1-2 句）

**反问不写入笔记正文**，只出现在回复中。使用者觉得有启发时，可另起一张回应笔记。

## 输出格式

```markdown
## 🔗 链接建议

**A · 语义相近**
✅ [[笔记名]] — 理由：[具体说明]（建议双向链接）
⬇️ [[笔记名]] — 关系模糊，建议单向链接并备注「待强化」
❌ [[笔记名]] — 丢弃原因：[泛泛相关，无法用具体关系描述]

**B · 时间邻近**
✅ [[笔记名]] — 理由：[同一思考线索]（双向链接已写入）

**⚡ C · 反相关**
✅ [[笔记名]] — 张力：[你在这里说X，那里说Y，可能有对话空间]（双向链接已写入）

## 🏷️ 关键词建议
- `词条A` → 建议添加到项目关键词总表的「X」词条下
  - **回答的问题**："当我想找[X]时"
- `词条B` → 已新建词条「Y」
  - **回答的问题**："当我想找[Y]时"

## 💬 Gegenrede
你说 [X]，但从 [Y 视角] 来看，[Z 质疑]？
```

## Validation Loop（做一步验一步）

仅在用户授权写入且写入完成后，执行以下验证循环：

### 轮次 1：链接可达性验证
对每条已写入的 `[[链接]]`：
1. **Glob 确认目标文件存在**（路径有效）
2. **确认反向链接已写入**（双向链接的目标笔记中包含指向当前笔记的 `[[]]`）

| 验证结果 | 动作 |
|---|---|
| 目标存在 + 反向已写 | ✅ 通过 |
| 目标存在 + 反向未写 | 立即补写反向链接 |
| 目标不存在 | 标记为「🔮 待创建」，链接保留但加注释 |

### 轮次 2：关键词总表一致性
1. 重读项目声明的关键词总表
2. 确认本轮写入的词条确实出现在总表中（防止写入失败/被覆盖）
3. 确认入口数符合目标：1–2 个；3 个可接受；4–5 个需收敛；>5 个触发警告

| 验证结果 | 动作 |
|---|---|
| 词条存在 + 指针正确 | ✅ 通过 |
| 词条缺失 | 重新写入 |
| 4–5 个入口 | 收敛或替换最弱入口 |
| >5 个入口 | 标记警告并执行拆词 / 合并入口方案 |

### 轮次 3：YAML 完整性
检查目标笔记的 frontmatter：
- `indexed: true` 已添加
- `links:` 字段与正文 `[[]]` 链接一致

### 退出标准
- 轮次 1-3 全部 ✅ → 输出验证摘要，结束
- 任一轮次有修复动作 → 修复后**重跑该轮次**，直到通过
- 最多循环 3 次（防止死循环），3 次仍未通过 → 输出失败项，暂停询问

### 验证摘要格式
```
✅ 验证通过（3/3）
- 链接可达：5/5 双向可达，0 待创建
- 关键词总表：2 词条已确认
- YAML：indexed + links 一致
```

---

## 质量检查

- [ ] 三路都已执行（A/B/C），总候选 3-5 条
- [ ] 至少 1 条 ⚡ 反相关（路径 C）
- [ ] 每条候选都有具体理由（不能只写"可能相关"）
- [ ] 关键词已按决策标准写入总表（追加/替换/新建），边缘情况已暂停询问
- [ ] 反问使用了与笔记**不同学科**的视角
- [ ] **已按三档裁定（接受/降级/丢弃）自主写入链接**，边缘情况已暂停询问
- [ ] **Validation Loop 3 轮全部通过**

## 完成标记

完成链接写入后，在笔记的 YAML frontmatter 添加:
```yaml
indexed: true
```

不再在文档末尾写"归网：✅"。
