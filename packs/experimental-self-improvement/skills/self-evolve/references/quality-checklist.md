# Self-Evolve 质量自检表

每轮进化的**设计阶段（Phase 1 尾声）**和**固化之后（Phase 3）**，请使用本列表进行自我审校，防止产生无效实验。

## 设计参数必备清单（必须全部明确）
- [ ] 基线状态已经完全量化
- [ ] 优劣判断指标已经确立
- [ ] `telemetry_hook`（`command` 与 `extract` 数组）已明确并在报告和 state.json 中准备好
- [ ] 每个方案观察的具体周期已定
- [ ] 方案切换点（若有）已精确到小时
- [ ] 提前终止条件（极大负面反应时的熔断条件）已明确
- [ ] 实验是可逆转的（随时可以回滚）
- [ ] **Harness 扩展**：若会改 Rule、Agent 配置、Skill、Hook 或质量 CI，`.agent-evolve/held-out-tasks.md` 中至少 **2 条 `[x]`** 已写好验收，或已接受「空集豁免」并会写 `HELD_OUT_SKIPPED_EMPTY`
- [ ] **用户授权**：拟固化的精确 diff、非只读命令和外部影响已预览；尚未把实验记录误当成执行授权
- [ ] **冷门配额**：已读 `.agent-evolve/self-evolve-meta.json`；若 `non_cold_streak >= 4`，本实验已标记为 `cold_skill_quota` 或已改选冷门路径
- [ ] **合成压测**（可选但推荐）：若瓶颈来自 `source: synthetic-stress`，变异与回放步骤已写在实验部署说明里

## 质量红线自检

| 维度 | 自检问题 | 不及格标准 |
|------|---------|-----------|
| **切口** | 进化方向是否足够小、足够具体？ | "提高整体效率" = 不及格 |
| **可测** | 成功指标是否可量化？ | "感觉更好/代码更整洁了" = 不及格 |
| **对比** | 有没有和基线做真正的对比？ | 没量化基线就直接宣布成功 = 不及格 |
| **可逆** | 如果方案失败，能回滚吗？ | 不可逆的深层架构变更未经人类确认 = 不及格 |
| **固化** | 赢家的结论是否已写入真正的配置文件？| 仅存在于实验日志中口头宣布未落地 = 不及格 |
| **Held-out** | 改 Agent 配置前是否跑过 `held-out-tasks.md` 的 `[x]` 项？ | 跳过门禁仍写入 Rule/Skills = 不及格（除非空集且已记 `HELD_OUT_SKIPPED_EMPTY`） |
| **授权** | 固化前是否展示精确改动并得到用户批准？ | 用实验状态代替授权，或自行安装、改 Hook/计划任务 = 不及格 |
| **冷门配额** | 连续非冷门是否超过 4 次仍注册非冷门？ | `non_cold_streak>=4` 仍注册非冷门 = 不及格 |
