# Task Routing — 任务分类与 Agent 动态路由

## 分类标准

### L1：战略 / 不可逆
- **特征**：影响公司方向、客户关系、对外形象；中途无法退出；错误代价高
- **判断问题**：「这个决策能在 24h 内完全撤回吗？」→ 不能 = L1
- **例子**：对外联络客户、发布公开内容、重大架构决策、签约/付款、修改全局 Agent 行为协议
- **路由**：→ 当前主 Agent 先明确选项和影响，取得用户确认后再执行

### L2：战术 / 部分可逆
- **特征**：影响一个项目或实验方向；可以调整但有成本；需要权衡取舍
- **判断问题**：「这个决策的影响范围超过一周吗？」→ 是 = L2
- **例子**：实验设计、内容策略、工具选型、skill 架构设计、进化实验启动
- **路由**：→ 当前主 Agent 与用户确认方向，再由可用的执行 Agent 或当前 Agent 落地

### L3：执行 / 高度可逆
- **特征**：具体执行任务；随时可以重做；影响范围小且局部
- **判断问题**：「做错了能在 1h 内修复吗？」→ 能 = L3
- **例子**：写代码、整理文件、搜索信息、写草稿、读文档、跑脚本
- **路由**：→ 当前 Agent 或已提供的专业执行 Agent 在现有授权范围内直接执行

---

## 路由决策树

```
任务进来
  ↓
能在 24h 内完全撤回？
  ├── 否 → L1 → 主 Agent 明确影响 → 用户确认 → 执行
  └── 是 → 影响范围超过一周？
              ├── 是 → L2 → 确认方向 → 可用执行 Agent 落地
              └── 否 → L3 → 当前或专业执行 Agent 直接执行
```

---

## Agent 路由表

| Agent | 擅长 | 适合任务类型 |
|-------|------|------------|
| 主 Agent | 决策、规划、协调、判断 | L1 影响说明、L2 方向确认 |
| 工程执行 Agent | 编码、文件操作、技术执行 | L3 技术执行、L2 实现落地 |
| 研究 Agent | 调研、信息整理、竞品分析 | L3 信息收集、L2 竞品/市场分析 |
| 写作 Agent | 写作、内容创作、文案 | L3 文案草稿、L2 内容策略 |

只使用当前 Runtime 实际提供的 Agent；不存在专业 Agent 时由主 Agent 执行，
不得虚构委派已经发生。

---

## 使用方式

收到任务时，先过一遍决策树，输出三行：

```
分类：L1 / L2 / L3
路由：推荐 [Agent 名]
理由：[一句话说明分类依据]
```

然后按路由建议执行或转交。
