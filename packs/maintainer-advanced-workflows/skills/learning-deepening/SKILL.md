# 学习深化：好奇大学生提问循环

## 角色定位

以**好奇大学生**的视角，通过多轮提问-补充材料的循环，将第一轮文献学习从"了解"深化到"理解"和"掌握"。

**核心使命**：将复杂技术/概念/架构的学习从表面理解深化到机制理解、关系理解、应用理解和深度理解。

## 触发条件

### 建议触发
- 第一轮学习完成后，如果材料涉及复杂技术、架构或概念，可以建议继续深化。
- 只有用户明确同意后才开始，不自动创建文档或修改原笔记。

### 手动触发
- 用户明确说"深化学习/提问循环/好奇大学生提问"
- 用户说"继续提问/再问几个问题"

## 输出契约

- **交付方式**：默认在对话中输出；用户要求落盘时才写入其指定项目路径
- **命名规范**：落盘时使用 `YYYYMMDD_[主题]学习深化_好奇大学生提问循环.md`
- **轮次要求**：至少3轮提问循环，默认3-5轮
- **问题质量**：每个问题都要有深度，能引发新思考
- **补充材料**：每个问题都要有对应的补充材料，包含定义、机制、示例
- **连接要求**：只有目标项目使用双向链接且用户授权修改原笔记时才建立
- **总结要求**：必须生成学习总结，列出核心洞察

## 工作流

### Step 0：前置检查

1. **确认第一轮文献笔记存在**
   - 检查是否有刚创建的文献笔记
   - 如果没有现成笔记，可使用用户提供的原始材料，并说明证据边界

2. **判断是否需要深化学习**
   - 检查主题复杂度和用户目标
   - 检查主题复杂度（技术/架构/概念）
   - 如果不符合，跳过深化学习

3. **读取第一轮文献笔记**
   - 读取完整的文献笔记内容
   - 提取关键概念、核心架构、重要设计决策

### Step 1：问题生成

1. **读取提问模板**
   - 读取`references/question-templates.md`
   - 了解好奇大学生角色设定和提问风格

2. **生成第一轮问题（3-5个）**
   - 基于第一轮文献笔记的关键概念
   - 聚焦机制理解（How）
   - 使用好奇大学生的提问风格
   - 确保问题有深度，不是表面问题

3. **问题质量检查**
   - 检查问题是否基于已学内容
   - 检查问题是否有深度
   - 检查问题是否聚焦核心概念
   - 检查问题是否与已提问的问题重复

### Step 2：材料补充

针对每个问题：

1. **搜索/分析材料**
   - 优先使用当前 Runtime 提供的项目搜索
   - 必要时使用web搜索补充
   - 引用原始文献笔记作为基础

2. **组织补充材料**
   - 定义：核心概念的精确定义
   - 工作机制：详细解释如何工作
   - 关键理解：用✅标记的关键洞察
   - 示例/类比：帮助理解的示例或类比
   - 应用场景：如何应用（如果是应用类问题）

3. **更新文档**
   - 默认在当前回答中组织问题和补充材料；只有用户已授权落盘时才追加文档
   - 使用标准格式（参考`references/output-template.md`）

### Step 3：循环判断

1. **判断是否需要继续**
   - 检查当前轮次（默认3-5轮）
   - 检查问题是否已充分覆盖核心概念
   - 检查用户是否明确说"停止"

2. **生成下一轮问题**
   - 基于已学内容和已提问的问题
   - 递进式深化：机制理解 → 关系理解 → 应用理解 → 深度理解
   - 避免重复已提问的问题

3. **继续循环**
   - 重复Step 2（材料补充）
   - 重复Step 3（循环判断）

### Step 4：总结生成

1. **生成学习总结**
   - 列出核心洞察（3-5条）
   - 总结通过N轮提问深化的理解
   - 提供下一步学习方向（可选）

2. **建立连接**
   - 若用户授权且目标项目使用双向链接，再与第一轮文献笔记建立连接
   - 未获修改授权时，只给出建议链接，不更新原笔记

3. **质量检查**
   - 检查是否至少3轮提问循环
   - 检查每个问题是否都有对应的补充材料
   - 检查补充材料是否包含：定义、机制、示例、应用
   - 检查是否建立了与原始文献笔记的连接
   - 检查是否生成了学习总结

## 提问生成策略

### 好奇大学生角色设定

**特点**：
- 聪明、好奇、不满足于表面理解
- 对"为什么"和"如何"有强烈的好奇心
- 关心"如何应用"和"需要注意什么"

**提问风格**：
- "我理解了X，但Y还是不太清楚..."
- "如果...会怎么样？"
- "为什么需要...？不能...吗？"
- "如果我要实现类似系统，最需要注意什么？"

### 问题类型递进

1. **第一轮：机制理解（How）**
   - 聚焦核心概念的工作机制
   - 关注组件之间的关系和交互
   - 追问设计决策的原因

2. **第二轮：关系理解（What vs What）**
   - 对比相似但不同的概念
   - 明确各自的职责边界
   - 理解它们如何协同工作

3. **第三轮：应用理解（How to Apply）**
   - 从理论转向实践
   - 关注实施难点和风险
   - 提供改进建议

4. **第四轮：深度理解（Why & What If）**
   - 追问第一性原理
   - 识别潜在问题和风险
   - 探索改进和优化方向

### 提问生成原则

1. **基于已学内容**：问题必须基于第一轮文献笔记中的关键概念
2. **递进式深化**：从机制理解到关系理解到应用理解到深度理解
3. **聚焦核心概念**：围绕核心架构、关键机制、重要设计决策
4. **避免重复**：检查已提问的问题，避免重复

## 材料补充方法

### 材料来源优先级

1. **当前项目搜索**（优先）
   - 使用当前 Runtime 可用的项目检索工具搜索相关概念
   - 查找相关文档和笔记

2. **原始文献笔记**（基础）
   - 引用第一轮文献笔记的内容
   - 基于文献笔记深化理解

3. **Web搜索**（补充）
   - 必要时使用web搜索补充
   - 查找官方文档、技术博客等

### 补充材料组织

每个问题的补充材料必须包含：

1. **定义**：核心概念的精确定义
2. **工作机制**：详细解释如何工作
3. **关键理解**：用✅标记的关键洞察
4. **示例/类比**：帮助理解的示例或类比
5. **应用场景**：如何应用（如果是应用类问题）

## 输出格式要求

### 文档结构

参考`references/output-template.md`的完整模板。

### 关键格式

1. **提问格式**：
   - 使用`## 🔍 第X轮提问`作为轮次标题
   - 每个问题使用`### 问题X：[问题标题]`
   - 问题内容使用引用格式：`> [问题内容]`

2. **补充材料格式**：
   - 使用`## 📚 补充材料X：[标题]`作为补充材料标题
   - 包含：定义、工作机制、关键理解、示例/类比
   - 使用表格、代码块、列表等格式增强可读性

3. **学习总结格式**：
   - 使用`## ✅ 学习总结`作为总结标题
   - 列出核心洞察（3-5条）
   - 提供下一步学习方向（可选）

## 质量检查清单

### 必须完成的检查项

- [ ] 至少3轮提问循环
- [ ] 每个问题都有对应的补充材料
- [ ] 补充材料包含：定义、机制、示例、应用
- [ ] 建立了与原始文献笔记的连接
- [ ] 生成了学习总结
- [ ] 问题有深度，不是表面问题
- [ ] 问题聚焦核心概念，避免发散
- [ ] 问题与已提问的问题不重复

### 避免的问题

- ❌ 过度提问：超过5轮，导致信息过载
- ❌ 问题发散：偏离核心概念
- ❌ 补充材料浅显：简单重复第一轮内容
- ❌ 缺少连接：没有与原始文献笔记建立连接

## 与 deep-learning 的集成

### 自动触发机制

1. **检测文献/结构笔记创建**
   - 在`deep-learning`完成后，检查是否有新创建的文献/结构笔记
   - 检查文献笔记的重要度（L2/L3）

2. **判断是否需要深化**
   - 复杂技术/架构/概念 → 需要深化
   - 简单概念/工具介绍 → 不需要深化

3. **自动启动深化学习**
   - 创建深化学习文档
   - 开始第一轮问题生成

### 文档连接

1. **深化学习文档 → 文献笔记**
   - 在深化学习文档的"关联文档"中链接文献笔记
   - 标注连接性质：**补充**、**深化**、**应用**

2. **文献笔记 → 深化学习文档**
   - 在文献笔记的"相关连接"中添加深化学习文档
   - 标注连接性质：**补充**、**深化**

## 循环终止条件

### 默认终止条件

- 完成3-5轮提问循环（默认）
- 问题已充分覆盖核心概念
- 用户明确说"停止"

### 可配置参数

- `maxRounds`：最大轮次数（默认5）
- `minRounds`：最小轮次数（默认3）

## 注意事项

1. **不要过度提问**：控制在3-5轮，避免信息过载
2. **保持聚焦**：问题应围绕核心概念，避免发散
3. **质量优先**：每个补充材料都要有深度，不是简单重复
4. **连接建立**：必须与原始文献笔记建立双向链接
5. **递进式深化**：每轮问题应该比上一轮更深入
