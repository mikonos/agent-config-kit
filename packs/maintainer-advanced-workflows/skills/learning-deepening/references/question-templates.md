# 提问模板与角色设定

## 好奇大学生角色设定

### 核心特点
- **聪明**：能理解复杂概念，但不满足于表面理解
- **好奇**：对"为什么"和"如何"有强烈的好奇心
- **不满足**：总是觉得"还差一点"，想要更深入的理解
- **实用导向**：关心"如何应用"和"需要注意什么"

### 提问风格特征

#### 1. 机制理解类
- "我理解了X，但Y还是不太清楚..."
- "X是怎么工作的？为什么这样设计？"
- "如果...会怎么样？"

#### 2. 关系理解类
- "X和Y是什么关系？区别是什么？"
- "为什么需要X和Y两个？不能合并吗？"
- "X在Y中扮演什么角色？"

#### 3. 应用理解类
- "如果我要实现类似系统，最需要注意什么？"
- "这个设计有什么局限性？"
- "如果我要改进，应该从哪里入手？"

#### 4. 深度理解类
- "这个设计的第一性原理是什么？"
- "有什么潜在风险？"
- "有没有更好的设计模式？"

## 问题类型分类

### 类型1：机制理解（How）
**目标**：理解工作原理和设计机制

**示例问题**：
- "Agent Runtime到底是什么？它和Gateway的区别是什么？"
- "Skills系统是怎么工作的？它和Agent Runtime是什么关系？"
- "Session是怎么管理的？如果会话历史太长怎么办？"

**生成策略**：
- 聚焦核心概念的工作机制
- 关注组件之间的关系和交互
- 追问设计决策的原因

### 类型2：关系理解（What vs What）
**目标**：理解不同概念/组件之间的关系和区别

**示例问题**：
- "Gateway的本地优先和云端大模型之间是什么关系？"
- "Session和Agent Loop是什么关系？"
- "Skills系统和节点系统有什么区别？"

**生成策略**：
- 对比相似但不同的概念
- 明确各自的职责边界
- 理解它们如何协同工作

### 类型3：应用理解（How to Apply）
**目标**：理解如何应用和需要注意什么

**示例问题**：
- "如果我要自己实现一个类似的系统，最核心的技术难点是什么？"
- "如果应用到当前项目，最需要注意的差异是什么？"
- "这个示例系统的设计有什么局限性？如果我要改进，应该从哪里入手？"

**生成策略**：
- 从理论转向实践
- 关注实施难点和风险
- 提供改进建议

### 类型4：深度理解（Why & What If）
**目标**：理解第一性原理、局限性和改进方向

**示例问题**：
- "这个示例系统的设计有什么局限性？如果我要改进，应该从哪里入手？"
- "如果用户量增加，性能会怎么变化？"
- "如果我要学习，应该从哪里开始？"

**生成策略**：
- 追问第一性原理
- 识别潜在问题和风险
- 探索改进和优化方向

## 提问生成原则

### 1. 基于已学内容
- 问题必须基于第一轮文献笔记中的关键概念
- 避免重复或过于浅显的问题
- 确保问题有深度，能引发新思考

### 2. 递进式深化
- 第一轮：机制理解（How）
- 第二轮：关系理解（What vs What）
- 第三轮：应用理解（How to Apply）
- 第四轮：深度理解（Why & What If）

### 3. 聚焦核心概念
- 围绕核心架构、关键机制、重要设计决策
- 避免发散到边缘话题
- 确保每个问题都有价值

### 4. 避免重复
- 检查已提问的问题，避免重复
- 如果问题相似，从不同角度提问
- 确保每轮问题都有新视角

## 提问模板

### 模板1：机制理解
```
### 问题X：[核心概念]是怎么工作的？

**大学生提问**：
> 我理解了[前置知识]，但[核心概念]还是不太清楚：
> 1. [核心概念]到底是什么？是[可能的误解1]还是[可能的误解2]？
> 2. 它是怎么工作的？[关键机制问题]
> 3. 为什么需要[核心概念]？不能[替代方案]吗？
```

### 模板2：关系理解
```
### 问题X：[概念A]和[概念B]是什么关系？

**大学生提问**：
> 我理解了[概念A]和[概念B]的基本概念，但还是不太清楚：
> 1. [概念A]和[概念B]的区别是什么？
> 2. 它们是什么关系？是[关系类型1]还是[关系类型2]？
> 3. 为什么需要两个？不能合并吗？
```

### 模板3：应用理解
```
### 问题X：如果我要[应用场景]，最需要注意什么？

**大学生提问**：
> 我理解了[理论/架构]，但我想知道：
> 1. 如果我要[应用场景]，最核心的[难点/注意点]是什么？
> 2. 需要哪些[前置条件/资源]？
> 3. 有什么[风险/挑战]？怎么应对？
```

### 模板4：深度理解
```
### 问题X：[主题]有什么局限性？如何改进？

**大学生提问**：
> 我理解了[主题]的设计，但我想知道：
> 1. 它有什么局限性？
> 2. 如果我要改进，应该从哪里入手？
> 3. 有没有更好的设计模式？
```

## 问题质量检查

### 检查清单
- [ ] 问题基于已学内容的关键概念
- [ ] 问题有深度，不是表面问题
- [ ] 问题能引发新思考，不是简单重复
- [ ] 问题聚焦核心概念，避免发散
- [ ] 问题符合当前轮次的问题类型
- [ ] 问题与已提问的问题不重复

### 避免的问题类型
- ❌ 过于浅显："什么是X？"（应该在第一轮文献笔记中已解答）
- ❌ 过于发散：偏离核心概念的问题
- ❌ 重复提问：与已提问的问题相似
- ❌ 无价值问题：不能引发新思考的问题
