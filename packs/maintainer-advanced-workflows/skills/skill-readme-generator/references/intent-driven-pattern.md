# Intent-Driven Pattern (Scenario-First)

## When to Use
Best for:
- Meta-tools (find-skills, skill-creator)
- Workflow orchestrators  
- Tools solving specific pain points

## Structure

```markdown
# [Skill Name]

## When to Use This Skill
Use this skill when the user:
- [Scenario 1: specific user pain point or question]
- [Scenario 2: another trigger context]
- [Scenario 3: user goal or workflow]
...

## What is [Core Concept/Tool]?
[Brief 2-3 sentence explanation]

[Code example showing basic usage]

## [Section with more details]
...
```

## Example from find-skills (#1 ranked, 172K+ installs)

```markdown
## When to Use This Skill
Use this skill when the user:
- Asks "how do I do X" where X might be a common task
- Says "find a skill for X" or "is there a skill for X"
- Asks "can you do X" where X is a specialized capability
- Express interest in extending agent capabilities
```

## Characteristics

✅ Leads with user scenarios (not features)  
✅ Natural language triggers (matches how users ask)  
✅ Immediately builds trust (user sees their exact problem)  
⚠️ Requires clear, distinguishable use cases

## Key Principles

- **Trigger language**: Quote actual phrases users might say
- **Problem-first**: Start with pain, not solution
- **Specificity**: Each scenario should be distinct and recognizable
- **No jargon**: Use natural language, not technical terms
