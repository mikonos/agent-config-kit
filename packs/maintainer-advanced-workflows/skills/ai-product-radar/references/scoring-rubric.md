# 评分体系（透明 · 可复现）

> 本文件是整个雷达的「裁判规则」。原 YouMind 版本用黑箱分数（"机会评分 1603 / 需求强度 1500"），
> 且分数与推荐自相矛盾。这里把每个输入都锚定到 0–N 刻度，机会分由**显式公式**算出，任何人可重算。

## 1. Phase 2：三重门筛选（Triple Gate）

原版核心判据：**高情绪强度 + 存在 workaround + 已有付费竞品**。任一不满足，信号降级为"观察池"，不进入评分。
之所以是闸门而非打分项：这三者缺一，机会要么是伪需求、要么无法变现、要么没人真的在解决。

| 闸门 | 通过标准 | 不通过的含义 |
|---|---|---|
| **G1 情绪强度** | 用户用了"ruined / useless / hate / 崩溃 / 再也"级词汇 | 只是"nice to have"，付费意愿弱 |
| **G2 Workaround 存在** | 用户正在用替代方案（纸笔 / ChatGPT / DIY 工具 / 物理隔离） | 没人真为它费力 = 痛得不够 |
| **G3 付费竞品验证** | App Store / 海外有人靠它收钱（哪怕做得烂） | 完全无竞品时，先标"蓝海待验证"，**不直接淘汰**——但需在报告里显式标注风险更高 |

> G3 特例：若"无付费竞品"恰恰因为是**全新 AI 品类**（如 AI 问责伙伴），按蓝海处理——稀缺度给高分，但在深档里要求 Layer 1 内容验证先行。

## 2. Phase 3：四轴评分（每轴有锚点）

### 轴一：需求强度 Demand（0–100）= 四个子项各 0–25 求和

| 子项 | 25（满） | 15（中） | 5（弱） |
|---|---|---|---|
| **情绪强度** | ruined/useless/hate 级 | frustrating/wish 级 | 轻微抱怨 |
| **频率/复现** | ≥5 个独立社区反复出现 | 2–3 个社区 | 仅 1 处 |
| **Workaround 强度** | 已付费买替代品 / 自建工具 | 手动替代（纸笔/计时器） | 仅抱怨无行动 |
| **支付暗示** | "已付费但不满 / 愿意付" | "wish it were affordable" | 无 |

> 子项之间独立打分再求和；不要让"情绪强度高"顺手抬高其它三项。

### 轴二：供给稀缺度 Scarcity（0–100）= 越高越无人服务

| 分档 | 标准 |
|---|---|
| 90–100 | 无任何 >$10K MRR 的产品命中该具体痛点（真蓝海） |
| 60–89 | 有竞品，但**全部存在同一个明确缺口**（= 你的差异化楔子） |
| 30–59 | 多个像样竞品；差异化可能但很窄 |
| 0–29 | 红海，成熟巨头垄断，难差异化 |

> 关键纪律：稀缺度衡量的是"**该具体痛点**有没有被很好解决"，不是"这个大类有没有 App"。习惯追踪是红海，但"断签后温和欢迎回来"可能仍是 70 分。

### 轴三：B2C 订阅制适配度 B2C-fit（0–10）= 三个子项求和（雷达气泡大小）

| 子项 | 满分 | 判据 |
|---|---|---|
| 价值复现性 | 4 | 需求是否每月反复发生（值得订阅而非一次性买断） |
| 情绪/身份溢价 | 3 | 用户是否愿为"成为某种人 / 摆脱某种焦虑"持续付费 |
| 可独立构建性 | 3 | 单人 / 小团队能否 3–7 天做出 MVP（边际成本低） |

### 轴四：AI 适配度 AI-fit（系数，雷达气泡颜色）

| 档 | 系数 | 判据 |
|---|---|---|
| 🟢 高 | ×1.0 | AI 是核心解锁：个性化对话 / 内容生成 / 状态感知，去掉 AI 产品不成立 |
| 🟡 中 | ×0.7 | AI 锦上添花，但 CRUD 版本也能用 |
| ⚪ 低 | ×0.4 | AI 只是装饰，普通工具就够（如纯计数器） |

## 3. 机会分公式与分档

```
机会分 = (Demand / 100) × (Scarcity / 100) × (B2C_fit / 10) × AI系数 × 1000
```

- 取值 0–1000，每一项都可追溯、可重算。
- 四个因子相乘（非相加）：任何一维塌方都拉垮整体——这正是机会的真相（红海里需求再高也不该 all-in）。

| 机会分 | 分档 | 行动 |
|---|---|---|
| ≥ 600 | 🟢 黄金 | 优先验证，本周启动 Layer 1 |
| 400–599 | 🟢 强 | 备选，资源到位即上 |
| 200–399 | 🟡 观望 | 差异化不够锐利，蹲点等信号变化 |
| < 200 | ⚪ 放弃 | 不投入 |

## 4. 雷达坐标映射（喂给 build_radar.py）

| 视觉通道 | 映射 | 量程 / 阈值 |
|---|---|---|
| X 轴 | Demand 需求强度 | 0–100，象限线 **70** |
| Y 轴 | Scarcity 供给稀缺度 | 0–100，象限线 **50** |
| 气泡大小 | B2C_fit | 0–10 |
| 气泡颜色 | AI-fit | 🟢高 / 🟡中 / ⚪低 |

四象限：右上=黄金机会区（高需求×高稀缺）｜右下=高需求低壁垒区（红海，需差异化）｜左上=高壁垒低需求区（教育市场）｜左下=观望区。

> 注意：原版雷达 Y 轴用 0–10、报告表却用 0–100，且把"无竞品蓝海"的 #8 画在 Y=38（低稀缺）位置——自相矛盾。本版统一 0–100，蓝海必然落在右上黄金区。

## 5. 一致性自检（必做）

评分完成后逐条核对，命中任一即返工：
1. **分数 ↔ 推荐一致**：机会分最高的，是否就是你在执行摘要里首推的？不一致必须解释或改分。
2. **稀缺度 ↔ 竞品现状一致**：标了"蓝海"的，稀缺度是否 ≥90？标了"红海"的是否 ≤29？
3. **气泡位置 ↔ 文字结论一致**：报告说"黄金机会"的点，是否真落在雷达右上象限？
4. **每个市场数字都有出处或认知状态标**（见 report-guide.md 的溯源纪律）。

## 6. 一个完整算例（#8 AI 问责伙伴）

- 需求强度：情绪 25（"what I was missing"）+ 频率 25（r/Habits、r/ADHD、r/getdisciplined 反复）+ Workaround 25（已用 ChatGPT 每日问责）+ 支付 20（"affordable way"）= **95**
- 供给稀缺度：现有都是 ChatGPT wrapper，无 >$10K MRR 真问责产品 → **90**（蓝海）
- B2C-fit：复现 4 + 情绪溢价 3 + 可独立构建 3 = **10**
- AI-fit：主动对话+状态感知=核心解锁 → 🟢 **×1.0**
- **机会分 = 0.95 × 0.90 × 1.0 × 1.0 × 1000 = 855 → 🟢 黄金**

对照 #2 ADHD 降负（稀缺度 55，因 Tiimo 等存在但有缺口；B2C 9）：`0.95×0.55×0.9×1.0×1000 = 470 → 🟢 强`。
→ #8 > #2，与"首推 #8"的结论一致。**原版正好相反，这就是透明公式的价值。**
