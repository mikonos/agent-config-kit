# Migration Guide（安全迁移清单）

## 通用原则
- **可回滚**：每一步都有撤销路径（或可接受的补救手段）
- **可观测**：上线前后能验证正确性与性能（指标/抽样/一致性检查）
- **分阶段**：扩展→回填→验证→切换→清理

## 典型安全模式
- **新增列（有默认值）**：避免一次性 `DEFAULT` 写全表；优先“先加可空列→回填→再加 NOT NULL/DEFAULT”
- **改名/删列**：双写或兼容期；先在应用层做兼容，再做 DB 变更
- **加索引**：在线建索引（按引擎能力）；评估锁表风险与耗时

## 最小上线检查
- [ ] 变更脚本可重复执行（幂等）或有防重入策略
- [ ] 评估锁表/写放大/磁盘空间
- [ ] 回滚方案与触发条件
- [ ] 验证方案：抽样、全量对账或校验和（按成本选择）
