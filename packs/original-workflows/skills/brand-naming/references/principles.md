# 小马宋命名法则详解（参考用）

> 执行命名任务时，核心在 SKILL.md 的「Step 0 先想清楚 + 两种方法 + 四步工作流 + 输出契约 + 首选理由」；本文件供需要展开理解或补充案例时按需查阅。

## 高效果执行要点（为何先想再取名）

- **先呈现「小马宋会怎么想」再给名字**，比直接丢候选表更易被采纳。
- **Step 0 必做**：谁买/谁用/谁在传 → 传播场景（怎么搜、怎么说）→ 品类文化母体与禁忌。双受众时显式写出「名字要同时通过谁的检验」。
- **发散用表格**：候选 | 母体/逻辑 | 备注，便于筛选与复现。
- **交付带首选理由**：对 Top 1 用 2～4 句话说明为什么排第一（传播成本、双受众、母体、带品类说法），并提醒商标与搜索排查。
- **更多候选**：扩展发散 → 合并去重清单（按字数分组）→ 提醒四维自检+商标检索；可选给第二梯队 Top 5～8。

## 核心法则 1～8（简要）

1. **降低传播成本**：名字最基本功能是让人记住；我们更容易记住「已经记住的东西」。检验：说一遍对方能正确写出来吗？
2. **听觉思维**：电话测试法（说一遍能写对吗）、输入法测试（搜狗无云存储一次性打出、不切换键盘）。
3. **文化母体**：非同类命名（小米手机、瓜子、豌豆荚）；不相容组合（蓝月亮、天猫、蓝小鱼、功夫恐龙）。小马宋实操：问创始人喜欢什么动物/颜色→快速收敛。
4. **陌生的熟悉感**：熟悉结构+新鲜元素（摇滚甜橙）；避免太熟无陌生感（小米面包）。
5. **名字与品类**：不误解（小米手机 OK，小米蛋糕误解）；传播时带品类（小米手机、滴滴打车、小狗电器）。
6. **为估值命名**：非同类命名比属性堆砌更有想象空间（瓜子/蚂蚁/天猫 vs 优信二手车/淘宝商城）。
7. **名字背后意义慎用**：受众不接收深层寓意，除非大众一听就懂。
8. **字数与气势**：多一字多一千万（段永平）；单字常补字（柒牌、郎酒）；节目名 4～5 字有气势。

## 五种不好的名字

不易记住、输入需切换、生僻字、改词改字、不用本国语言。——详见 SKILL.md 主表。

## 人名 / 花名

原则同品牌：好记好念。王菲 vs 王靖雯；花名破冰（小马宋、成龙、脱不花）、破等级感（阿里：风清扬、逍遥子）。

## 双受众命名示例（购买者≠使用者）

**场景**：面向中国老年人的智能风险识别产品（如步态防跌倒），购买者=子女，使用者=老人。

- **Step 0 要点**：名字要子女愿说（「我给我爸妈买了个 XX」）、老人不抵触（不显监控/医疗）；传播时带品类自然（「XX 守护」「爸妈用的 XX」）。文化母体用安心/灯/燕子/屋檐，避免监护/监测感。
- **候选示例**：安心燕（燕子+安心）、一盏灯（给爸妈装一盏灯）、稳稳（叠词+稳）。首选理由可写：安心燕—国民级燕子意象、安心打中子女动机、说一遍能写对、有画面；建议先做商标与搜索排查。
