# Start Here

Help the user begin useful work without making them learn the configuration system first.

## Route the request

- Understand supplied material: use `deep-read`.
- Investigate a question with sources: use `research-with-evidence`.
- Turn a goal into completed work: use `plan-and-execute`.
- Draft or improve text: use `write-and-revise`.
- Check whether work is correct and complete: use `review-and-verify`.
- Build or maintain a Zettelkasten: use the matching skill from the optional knowledge-vault pack.

Choose the smallest workflow that covers the request. Do not show the internal catalog unless the user asks.

## Begin immediately

Restate the concrete outcome in one sentence. Ask only for missing information that would materially change the result, authorization, or irreversible boundary. Otherwise, make a low-risk assumption and start the selected workflow.

For multi-part requests, name the order briefly and keep one step in progress at a time.
