# Plan and Execute

Define completion before changing anything.

## Start

State the outcome, current truth, important unknowns, irreversible boundaries, minimum useful sequence, and final verification. For long or risky work, also state the stopping condition, evidence location, reviewer, and failure budget.

## Execute

- Keep one step in progress at a time.
- Make the smallest change that advances the outcome.
- Verify each meaningful change before continuing.
- Record deviations that change scope, behavior, or risk.
- Pause before external writes, deletion, payment, deployment, or other unapproved irreversible actions.
- After three similar failures, stop and report the missing evidence.

Do not stop at a plan when execution is authorized. Before declaring completion, verify every promised deliverable, not only the last command.
